# Настройка Git репозитория

## ✅ Локальный репозиторий готов

Проект уже инициализирован и первый коммит создан.

## Загрузка в удаленный репозиторий

### Вариант 1: GitHub

1. Создайте новый репозиторий на GitHub:
   - Перейдите на https://github.com/new
   - Введите название репозитория (например: `sellio`)
   - **НЕ** создавайте README, .gitignore или лицензию (они уже есть)
   - Нажмите "Create repository"

2. Подключите удаленный репозиторий:
   ```bash
   git remote add origin https://github.com/ВАШ_USERNAME/sellio.git
   ```

3. Загрузите код:
   ```bash
   git branch -M main
   git push -u origin main
   ```

### Вариант 2: GitLab

1. Создайте новый проект на GitLab

2. Подключите удаленный репозиторий:
   ```bash
   git remote add origin https://gitlab.com/ВАШ_USERNAME/sellio.git
   ```

3. Загрузите код:
   ```bash
   git push -u origin main
   ```

### Вариант 3: Другой Git хостинг

1. Создайте репозиторий на вашем хостинге

2. Подключите удаленный репозиторий:
   ```bash
   git remote add origin URL_ВАШЕГО_РЕПОЗИТОРИЯ
   ```

3. Загрузите код:
   ```bash
   git push -u origin main
   ```

## Проверка статуса

Проверить текущее состояние:
```bash
git status
```

Просмотреть историю коммитов:
```bash
git log
```

Просмотреть подключенные удаленные репозитории:
```bash
git remote -v
```

## Дальнейшая работа

После настройки удаленного репозитория, для отправки изменений используйте:

```bash
git add .
git commit -m "Описание изменений"
git push
```

## Важные файлы в .gitignore

Следующие файлы и папки автоматически исключены из репозитория:
- `node_modules/` - зависимости
- `.env` - переменные окружения (содержат секреты!)
- `dist/`, `build/` - скомпилированные файлы
- `.next/` - Next.js build
- `*.log` - логи
- `uploads/` - загруженные файлы

**Важно:** Не забудьте создать файлы `.env` на сервере/у других разработчиков на основе `.env.example`!
